# SALES-READY RELEASE — Definition and Criteria

Sales-ready means the buyer can use the product against real data with clear expectations.

## Mandatory criteria

- real Supabase authentication works.
- implemented CRUD persists to the database.
- deployment steps are documented.
- browser QA is signed.
- unsupported features are disclosed before sale.
- Arabic and RTL layouts are device-tested.
- backup and recovery path is defined.

## v1.0 sales positioning

Sell v1.0 as a hosted single-center spa management app for daily operations.

Do not sell v1.0 as complete POS/accounting, offline desktop software, or multi-customer SaaS.

## v1.1 sales expansion

Advertise checkout, financial reports, receipt print, settings mutations, and customer history only after v1.1 is implemented and verified.

## v2.0 sales expansion

Windows Desktop EXE becomes a separate offer after Tauri, SQLite, local auth, local backup/export, print, and Windows smoke tests are complete.
